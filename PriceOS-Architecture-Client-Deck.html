Section 1 Executive Summary

A Revenue Manager selects a property and date range → clicks "Run Aria" → AI agents scan the internet for competitor rates and events → the manager chats with Aria (the AI Revenue Manager) to get pricing optimization proposals → reviews and approves them.

PriceOS is an AI-powered revenue management platform designed for Dubai's short-term rental market. It combines real-time internet research, multi-agent AI analysis, and safety guardrails to help property managers optimize nightly pricing.

Section 6 The Pricing Formula

Step 1: Compute Base Price

For each available date:

  IF date overlaps a market_event → base = benchmark.recommended_event
  ELSE IF Friday or Saturday      → base = benchmark.recommended_weekend
  ELSE (weekday)                   → base = benchmark.recommended_weekday

  FALLBACK (if benchmark = 0):
    base = MAX(current_price, floor_price)
    Weekend: base × 1.10  |  Event: base × 1.20

Step 2: Apply Multipliers

Factor Condition Multiplier
Event Impact High impact × 1.30
Medium impact × 1.15
Low impact × 1.05
Occupancy Below 30% × 0.90 (low demand)
Above 70% × 1.10 (high demand) 
proposed_price = ROUND(base × event_factor × occupancy_factor)

Step 3: Clamp to Guardrails

if proposed_price < floor_price  → proposed_price = floor_price
if proposed_price > ceiling_price → proposed_price = ceiling_price

Mandatory This step has NO exceptions.

Step 4: Validate & Assign Risk

Rule Check If Fails
Rule 1 price ≥ floor? REJECT
Rule 2 price ≤ ceiling? REJECT
Rule 3 |change| ≤ 50%? REJECT
Rule 4 change > 25% has event? FLAG
Rule 5 price < P25? FLAG
Rule 6 price > P75? FLAG
All pass — APPROVED

Example Calculation

Property: floor=600, ceiling=1800, current=400, occupancy=47%

Date Day Type Base Event Proposed After Clamp
Mar 5 (Wed) Weekday 420 — 420 600 ↑ clamped to floor
Mar 7 (Fri) Weekend 550 — 550 600 ↑ clamped to floor
Mar 8 (Sat) Event + Weekend 700 ×1.30 910 910 ✅
Mar 12 (Wed) Weekday 420 — 420 600 ↑ clamped to floor

Section 7 Safety & Guardrails — Defense in Depth

PriceOS uses a three-layer defense system. Even if one layer fails, the others catch it:

Layer 1: Prompt Instructions LLM Instructions

PriceGuard prompt explicitly says "NEVER below floor price." CRO Router is told "always delegate pricing to PriceGuard."

✅ Usually works    ⚠️ LLM can sometimes hallucinate or ignore

Layer 2: PriceGuard Agent Dedicated AI Validator

A separate agent with temp=0.0 (zero creativity). Runs a strict 4-step validation pipeline. Has unconditional veto power — even Aria cannot override it.

✅ Very reliable    ⚠️ Still an LLM at its core

Layer 3: Server-Side Code Hardcoded TypeScript

The enforceGuardrails() function runs in TypeScript AFTER the LLM responds. It performs floor/ceiling clamping, swing checks (±50%), and deterministic risk calculation. Cannot be bypassed by any prompt change.

✅ ALWAYS runs    ✅ Deterministic    ✅ Testable

Section 21 FAQ — Frequently Asked Questions

🛡️ Can the AI ever set a price below my floor price?

No — it is physically impossible. The floor price is enforced by three layers: (1) AI prompt, (2) PriceGuard agent, and (3) server-side TypeScript code. Even if layers 1 and 2 fail, layer 3 always runs.

📊 How accurate are the competitor rates?

They come from live internet searches of Airbnb and Booking.com. In well-supplied areas like JBR, Marina, Downtown, accuracy is ±5-10%. Always review the "comps" list to verify.

🔄 If I approve a price, does it update on Airbnb automatically?

Not yet. Currently manual. Auto-push to Hostaway is the next major feature (~3 weeks). See Section 10.

👥 Can my team of 5 use this simultaneously?

Yes. Each user has independent sessions. Two managers can analyze different properties at the same time.

💰 How much does it cost per property?

AI costs are ~$1-2 per property/month with typical usage. Infrastructure is ~$25-50/month shared. See Section 13.

📱 Does it work on mobile/tablet?

Responsive on tablets (iPad). For phones, the layout collapses but laptop/tablet is recommended for analysis.

🔌 What PMS systems are supported?

Currently Hostaway only. Guesty and Lodgify planned. Custom REST API connectors available on request.

⚡ What if Aria gives wrong prices?

Server-side code always clamps prices to your floor/ceiling. Bad proposals are auto-REJECTED. You always have final approval.
