01 / Platform Overview

PriceOS is a multi-tenant short-term rental revenue management platform powered by a fleet of specialised Lyzr Studio agents, each scoped to a single purpose and communicating exclusively through structured MongoDB reads.

Key Architecture Principle: No agent ever uses stale global context. At every query, the system reads live, real-time data from MongoDB — scoped by orgId — builds a structured JSON context block, and injects it into the Lyzr agent's prompt before forwarding the user's question. This makes every answer deterministic and tenant-isolated.

Mode Detection Logic: When the onboarding completes, the server inspects the listing IDs saved in the session. If any listing ID starts with "demo-", Demo Mode activates. Otherwise, Live Mode runs and invokes Lyzr agents.

06 / db-context-builder.ts — How Context is Built

This server-side function is executed on every agent API call. It performs 5 MongoDB reads in ~100ms and assembles the JSON block injected into every Lyzr prompt.

MongoDB Reads Executed (Property Mode)
// 1. Validate ownership + fetch property metadata
Listing.findOne({ _id: listingId, orgId })

// 2. Fetch next 30-day calendar
InventoryMaster.find({
  listingId,
  date: { $gte: today, $lte: +30days }
}).sort({ date: 1 })

// 3. Fetch active bookings in window
Reservation.find({
  listingId,
  status: "confirmed",
  $or: [{ checkIn: { $lte: endStr }, checkOut: { $gte: startStr } }]
})

// 4. Fetch enabled pricing rules
PricingRule.find({ listingId, enabled: true })

// 5. Fetch upcoming market events (org-scoped)
MarketEvent.find({
  orgId,
  endDate: { $gte: today },
  startDate: { $lte: +30days },
  isActive: true
})
Why this matters: The agent never hallucinates pricing data because every number it references (occupancy %, current prices, booked nights, event premium) comes directly from the live database query result injected into its prompt. The agent is instructed: "TRUST THE FIGURES BELOW EXCLUSIVELY."

07 / Security & Tenant Isolation

Every database access is hardened against cross-tenant data leakage through three enforcement layers.

🔒
Layer 1: JWT Authentication

Every API request validates a signed JWT. The orgId is extracted from the token — not from user-supplied query parameters. Forged orgIds are impossible.

🗃️
Layer 2: orgId Filter on Every Query

Every MongoDB query includes { orgId: ObjectId(session.orgId) }. A user from Organisation A cannot read listings, inventory, or events belonging to Organisation B — even if they know the document ID.

🤖
Layer 3: Agent Context Scoping

Lyzr agents never receive raw database dumps. They only see the pre-filtered context JSON built by buildAgentContext(orgId). The agent cannot ask the database anything — it can only reason over what was injected.

finalMessage = "[SYSTEM CONTEXT]\n" + JSON.stringify(dbContext) + "\n\n[USER QUESTION]\n" + userMessage
This combined string is sent as the message field to the Lyzr Studio Chat API (LYZR_API_URL) with x-api-key: LYZR_API_KEY.
