# Collection: `insights`

**Topic:** Revenue Management Alerts & Recommendations
**Agent Access:** READ / WRITE (PriceGuard, Guardrails, CRO Router)

## Description
This collection stores actionable insights generated by the system. These are displayed as "cards" on the user dashboard. Each document represents a specific opportunity or risk identified in the data.

## Schema & Fields

| Field | Type | Semantic Meaning |
|---|---|---|
| `listingId` | ObjectId | References `listings._id`. |
| `category` | Enum | `pricing`, `occupancy`, `event`, `competitor`. |
| `title` | String | Short headline (e.g., "Critical: You are 40% below market median"). |
| `body` | String | Full explanation and suggested strategy. |
| `severity` | Enum | `critical` (immediate action), `warning`, `info`. |
| `status` | Enum | `new`, `seen`, `dismissed`. |
| `actions` | Object[] | UI buttons (e.g., `label: "Apply Price Change", action: "NAVIGATE_TO_RULES"`). |

## Relationship Logic
- **Alert Trigger:** `PriceGuard` writes to this collection when it flags a pricing anomaly.
- **Decision Support:** The `CRO Router` reads these to highlight priorities when the property manager logs in.
