PriceOS Intelligence Architecture Complete Map

Complete technical specification of exactly which MongoDB collections are read (or written) by each of the 10 System Agents to generate accurate, real-world behaviours.

Security Context: All database queries made by these agents are tightly scoped via the orgId of the currently authenticated user. All inference-time chat agents retrieve context strictly via db-context-builder.ts, preventing multi-tenant data bleed. System and infrastructure tables are completely sealed off from AI context.

Agent Semantic Data Dictionary

Query Context: Below are the raw fields and exact MongoDB / Mongoose JSON queries that the context-builders and background agents evaluate to assemble accurate AI context prompts. Since the transition from SQL, all data relies strictly on NoSQL MongoDB architecture.

1. Listing
The master property registry. Every short-term rental property managed by PriceOS is stored here. Each document represents one physical property. Use priceFloor and priceCeiling explicitly to ensure your pricing bounds are respected.
  • _id & orgId
    The primary key and Organization tenant isolator.
    db.listings.find({ orgId: ObjectId("...") })
  • hostawayId & name
    PMS identifier and the human-readable string.
    db.listings.find({ name: { $regex: /marina/i } })
  • bedroomsNumber & personCapacity
    Physical integers restricting booking parties.
    db.listings.find({ bedroomsNumber: { $gte: 2 } })
  • price, priceFloor, priceCeiling
    Base rate and strict AI boundaries in standard currency.
    db.listings.find({ priceFloor: { $lte: 500 } })
2. InventoryMaster
The daily operations and pricing matrix. It consolidates both the current calendar state (booked, blocked, available) and the AI's future pricing proposals in a single row per day per listing.
  • listingId & date
    Date string (YYYY-MM-DD) mapping isolated to the parent Listing.
    db.inventorymasters.find({ listingId: ObjectId("..."), date: { $gte: "2026-03-01", $lte: "2026-03-31" } })
  • status
    Enum string matching "available", "booked", "blocked".
    db.inventorymasters.find({ status: "available" })
  • currentPrice & proposedPrice
    Live price upstream and the AI engine's proposed change.
    db.inventorymasters.find({ proposedPrice: { $ne: null } })
  • proposalStatus & reasoning
    AI status gate loop ('pending', 'approved') + justification text output.
    db.inventorymasters.updateMany({ proposalStatus: "pending" }, { $set: { proposalStatus: "approved" } })
3. Reservation
The guest booking and financial ledger. Each row represents one guest reservation synced from Hostaway. Utilized primarily by Booking Intelligence for financial math.
  • guestName & channelName
    PII strings mapping client names to their origin booking site (e.g. Airbnb).
    db.reservations.find({ channelName: "Airbnb" })
  • status & totalPrice
    Enum ("new", "modified", "cancelled") and float value of entire revenue sum.
    db.reservations.aggregate([{ $match: { status: "new" } }, { $group: { _id: "$listingId", totalRevenue: { $sum: "$totalPrice" } } }])
  • checkIn & checkOut
    Date strings verifying when the financial sum was realized.
    db.reservations.find({ checkIn: { $gte: "2026-03-01" } })
4. MarketEvent
Scraped holidays, conferences, and mass events shaping demand. Data injected directly into context to justify premium price hikes to users.
  • name & description
    String representations (e.g. "Dubai Airshow") and AI contextual data blocks.
    db.marketevents.find({ name: { $regex: /airshow/i } })
  • impactLevel & upliftPct
    Enum severity limit ('moderate', 'high', 'critical') and exact mathematical percentage to inflate prices by.
    db.marketevents.find({ impactLevel: "critical", upliftPct: { $gte: 25 } })
  • startDate & endDate
    String bounds restricting when the given percentage bump is legally allowed to execute.
    db.marketevents.find({ startDate: { $lte: "2026-11-20" }, endDate: { $gte: "2026-11-20" } })
5. PricingRule
User-defined restrictions and limits over overriding behaviour such as DOW buffers, last-minute discounts, or blockout phases. Validates against PriceGuard constraints.
  • ruleType & priority
    Evaluation tree logic constraint ('DOW', 'LEAD_TIME') and integer stack resolution priority.
    db.pricingrules.find({ ruleType: "DOW" }).sort({ priority: 1 })
  • priceAdjPct & daysOfWeek
    Mathematical drop value (-10) and array restriction mapping for weekends e.g. [4, 5].
    db.pricingrules.find({ daysOfWeek: { $in: [4, 5] }, priceAdjPct: { $lt: 0 } })
6. BenchmarkData
Competitive Market Scanner data structure mapped across AirDNA/OTA comparisons. Drives the base intelligence of the Optimization Agents.
  • p25_rate, p50_rate, p75_rate
    Exact 25th, 50th (median), and 75th percentile statistical anchors locally sourced.
    db.benchmarkdatas.find({ p50_rate: { $gt: 1000 } })
  • percentile & verdict
    Calculated system posture e.g. 'OVERPRICED', 'UNDERPRICED'.
    db.benchmarkdatas.find({ verdict: "OVERPRICED" })
7. Insight
Anomaly warning and High-Risk Notification arrays blasted to the Dashboard by background execution monitors.
  • type & severity
    The failure vector ('occupancy_drop') and threat risk ('critical', 'high').
    db.insights.find({ severity: "critical", isDismissed: false })
8. HostawayConversation & GuestSummary
Raw guest inbox string logs and AI generated text outputs compressing conversation flows prior to dispatch.
  • summaryText & actionItems
    Processed AI contextual output logs and task strings.
    db.guestsummaries.find({ "actionItems.length": { $gt: 0 } })
